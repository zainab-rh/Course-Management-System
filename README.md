# Course Management System

A third-year Computer Engineering project developed for the **Software Engineering** course. This application provides a robust administrative solution for university resource management, focusing on the dynamic allocation of faculty, facilities, and student enrollments.

---

## Project Overview
The **Course Management System** was designed to solve the logistical challenges of university administration. The primary objective was to engineer an efficient system that minimizes scheduling conflicts and capacity issues while streamlining manual administrative workflows.

### Core Objectives
* **Conflict Resolution:** Minimize overlaps in classroom and faculty scheduling.
* **Resource Optimization:** Efficiently assign classrooms based on capacity and course requirements.
* **Operational Efficiency:** Reduce manual administrative overhead through automated enrollment and assignment logic.

---

##  Key Features
* **Faculty Allocation:** Dynamically assign instructors to specific academic courses.
* **Infrastructure Management:** Allocate classrooms to courses with automated capacity validation.
* **Student Lifecycle:** Handle full student enrollment flows for available academic terms.
* **Data Visualization:** Real-time lookup of enrollment lists and course rosters.

---

## Technical Stack
* **Framework:** .NET Framework
* **Development Environment:** Microsoft Visual Studio
* **Engineering Principles:** Object-Oriented Programming (OOP), Conflict Minimization Logic, and Software Lifecycle Management.

---

## Setup & Prerequisites
To run this project locally, ensure you have the following installed:

* **Open Solution:** Launch the `.sln` file using **Microsoft Visual Studio**.
* **Build & Run:** Rebuild the solution to resolve dependencies and execute the application.
